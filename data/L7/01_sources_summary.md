# Источники данных метеопроекта (выжимка для L7)

## Операционная БД (семинар L6)

**Источник для L7–L9** — **PostgreSQL команды** после семинара **L6**.

Структуру таблиц **не дублируем в раздаче** — смотрите в своей БД и фиксируйте в карте слоёв.

| Компонент | Назначение |
|-----------|------------|
| **PostgreSQL (L6)** | Operational: данные после загрузки из S3 |

## Потоки

```
Семинар L6 (operational PostgreSQL)
         ↓ batch ETL (L8, схема dwh)
RAW → ODS → DDS → DM → дашборд / отчёты
```

## Логическая модель проекта (L7)

На L7 проектируем **целевую** модель — телеметрия, прогнозы, витрина «точность прогноза». Словарь: `00_glossary_ru.md`, `CONVENTIONS.md`.

| Слой | Артефакт (целевая модель) |
|------|---------------------------|
| ODS | `ref_*`, `telemetry`, `forecasts`, … |
| DDS | `fct_measurement`, `fct_forecast`, `fct_forecast_error` |
| DM | `fct_forecast_accuracy_daily` + dims |

**Lab L8** — упрощённый pipeline: L6 → RAW → ODS → **`dm.fct_daily_weather`** (без отдельного DDS). См. семинар L8.

## FR аналитики

> Ошибка прогноза по **станциям / регионам / провайдерам / метрикам / дням** — целевая модель L7.  
> На lab L8/L9 — суточные наблюдения в `dm.fct_daily_weather` (DQ и KPI на L9).
