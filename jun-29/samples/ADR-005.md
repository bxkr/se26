# ADR-005: Механизм уведомлений оператора и администратора о сбоях

**Статус:** Принят  
**Дата:** 29.06.2026  
**Связано с требованием:** US-1 (нет прогноза/измерения → уведомить администратора), US-2 (некорректное измерение → уведомить администратора), US-3/US-4/US-6 (сервер упал → сообщить оператору), US-5 «Оператор управляет сервером».

### Контекст

Пять из шести user story содержат негативный сценарий с уведомлением оператора или администратора. Уведомления отправляются из разных сервисов (сервис приёма телеметрии, API, сервис прогнозирования). Если каждый сервис шлёт уведомления напрямую (email/Telegram), логика рассылки дублируется, а при отказе самого сервиса уведомление не уйдёт.

### Решение

Выделяем отдельный Notification Service, подписанный на топик `alerts` в Kafka. Каждый сервис при сбое публикует событие в `alerts`; Notification Service доставляет его по email и/или Telegram. Kafka гарантирует, что событие не потеряется, даже если Notification Service временно недоступен.

### Рассмотренные альтернативы

- **Прямая отправка email из каждого сервиса** — отвергнуто: логика уведомлений дублируется во всех сервисах; при отказе сервиса в момент сбоя уведомление не уйдёт; сложно централизованно менять каналы доставки.
- **Периодический polling статус-страницы оператором** — отвергнуто: не соответствует US-5 (оператор должен получать уведомления, а не сам опрашивать систему); задержка реакции неприемлема.
- **Прямая интеграция с PagerDuty / внешней on-call платформой** — отвергнуто на текущем этапе: избыточно для 500 станций и небольшой команды; можно добавить позже поверх Notification Service.

### Последствия

- **Плюсы:** единая точка управления каналами и шаблонами уведомлений; события не теряются при перезапуске Notification Service (Kafka хранит офсеты); легко добавить новый канал доставки.
- **Минусы / цена:** зависимость от Kafka (уже принята в ADR-001); небольшая задержка уведомления = время transit через брокер; Notification Service — ещё один компонент, который сам может упасть (нужен мониторинг его работоспособности).
